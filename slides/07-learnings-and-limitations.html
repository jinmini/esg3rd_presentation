개발 과정에서 마주한 3가지 핵심 제약사항

데이터 접근성 한계
Problem Statement
실시간 정량적 기업 데이터(에너지 사용량, 폐기물 배출량 등)에 대한 접근이 제한되어, 공개된 텍스트 데이터 중심의 정성적 분석에 집중할 수밖에 없었습니다.
Key Learning
진정한 ESG 보고 도구의 전제조건은 견고하고 신뢰할 수 있는 정량적 데이터 파이프라인 구축임을 깨달았습니다.
프로세스 통합 부족
Problem Statement
'중요성 평가 엔진'(슬라이드 4)과 'GRI Answer Engine'(슬라이드 5) 간의 데이터 흐름이 아직 완전히 자동화되지 않았습니다.
Key Learning
서로 다른 분석 모듈 간 매끄러운 사용자 경험 구현을 위해서는 표준화된 데이터 스키마의 중요성을 인식했습니다.
컴퓨팅 자원 제약
Problem Statement
KoAlpaca-5.8B와 같은 대규모 언어모델의 로컬 파인튜닝과 서빙에는 상당한 GPU 자원과 시간이 필요해 확장성에 제약이 있었습니다.
Key Learning
실용적인 상용 서비스를 위해서는 MLOps 인프라, 클라우드 기반 서빙, 그리고 모델 최적화 기법(양자화/증류)의 필요성을 이해했습니다.
